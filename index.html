Methodology

Each probe location runs two tests hourly:

Direct connectivity — TCP connection attempts to all five Telegram production datacenters on port 443. If these fail, Telegram is blocked at the network level from that location.

Proxy E2E — A real Telegram client connects through a Teleproxy instance using fake-TLS transport, authenticates, and makes an API call. This proves the full circumvention path works end-to-end.

If direct connectivity fails but the proxy test succeeds, Telegram is blocked but accessible through Teleproxy.
